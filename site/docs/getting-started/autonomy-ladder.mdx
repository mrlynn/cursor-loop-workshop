Jumping straight to full autonomy burns trust: agents over-build, humans lose visibility, and one bad demo reinforces prompt mode. The autonomy ladder maps tasks to Cursor surfaces so you progress deliberately.

## Why this matters

The framework tags every corpus task with the rung it teaches. Curriculum walks learners up the ladder; facilitators match demos to the room's comfort level. Knowing your rung tells you which rules, verifiers, and human checkpoints to apply.

## What you'll learn

- You can place each Cursor surface on rungs 0–3
- You can choose the right rung for a given task and audience
- You can describe the human's role at each rung
- You can plan a pre/post workshop rung movement goal

## How it works

| Rung | Mode | Verifier | Human role |
|------|------|----------|------------|
| **0** | Prompt-only (Chat / Cmd+K) | None — human eyeballs | Closes the loop |
| **1** | Agent mode, supervised | Single test file | Approves each iteration |
| **2** | Agent mode + loop rules | Test suite + lint | Reviews at green |
| **3** | Background / Cloud agent | Full verifier + budget | Reviews the PR |

Rung 0 is not wrong — Tab and Inline Edit remain the right tools for many edits. The ladder teaches **when to escalate** to a loop, not to replace all prompting.

At rung 1, the human still approves terminal commands and each iteration. At rung 2, `.cursor/rules` encode act→verify→correct→repeat; the human steps in at green or when blocked. At rung 3, the loop runs asynchronously in a worktree; the human reviews the resulting PR.

<LoopFeedbackDiagram />

The rungs below map to stages on the [Developer Journey](./developer-journey) continuum — from Chat/Composer (Rung 0) through Cloud agents (Rung 3).

## When to use this

| Your situation | Start at | Escalate when |
|----------------|----------|---------------|
| Skeptical audience, first demo | Rung 0 vs 1 side-by-side | They ask "can it run the tests itself?" |
| Hands-on lab with facilitators | Rung 1–2 | Learner has run one supervised loop |
| Internal dogfood on known tasks | Rung 2 | Verifier stable, rules committed |
| Long-running fix, low urgency | Rung 3 | Budget and Hooks configured |

## Common mistakes

### Skipping rung 1 in live demos
**Symptom:** Audience never sees the cognitive load difference between prompt and loop.  
**Fix:** Always run arm A (prompt-only) before arm B in comparison demos.

### Rung 3 without budget
**Symptom:** Background agent runs overnight, surprise diffs, trust collapse.  
**Fix:** Set max iterations and termination rules before launching background runs.

### Wrong rung for task difficulty
**Symptom:** Rung 2 rules on a task that needs human judgment every step.  
**Fix:** Lower the rung or improve the verifier with `@verifier-crafting`.
