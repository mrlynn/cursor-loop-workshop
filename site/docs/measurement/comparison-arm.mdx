The comparison arm is the workshop's proof moment. Run it live with `/run-comparison` and narrate human interventions, not just pass rate.

## Why this matters

User story from PRD §10: "As a developer new to agents, I want to see the same task done prompt-only and as a loop so I can judge whether the loop is worth it for my work." This page is that story's script.

## What you'll learn

- You can facilitate arm A without leaking verifier into the prompt
- You can facilitate arm B with rules + skill + full task spec
- You can optionally demo arm C with background agent + PR review
- You can log comparable JSON for each arm

## How it works

**Arm A — Prompt-only**

1. Fresh Chat thread (not Agent).
2. Goal only: "Fix the order status race so concurrent updates stay consistent."
3. No test command in prompt.
4. Human applies suggested patch.
5. Human runs `pytest tests/order/test_status_transition.py`.
6. Log pass/fail and intervention count.

**Arm B — Agent loop**

1. Fresh Agent thread.
2. `@loop-discipline @loop-termination @loop-safety`
3. `/loop-engineering` + full spec with verifier.
4. Agent iterates until green or blocked.
5. Log iterations, regressions, interventions.

**Arm C — Background** (optional)

:::caution VERIFY
Confirm Background Agent launch UX before live demo.
:::

1. Same prompt as B via Cloud/Background.
2. Human reviews PR.
3. Run verifier on branch.

## When to use this

| Cohort profile | Arms to run | Time |
|----------------|-------------|------|
| Skeptical, first hour | A + B | 25 min |
| Experienced with Agent | B + C | 30 min |
| Full workshop | A + B + C | 45 min |

## Common mistakes

### Verifier hint in arm A
**Symptom:** A and B look the same.  
**Fix:** Arm A prompt has no test command.

### No git reset between arms
**Symptom:** B starts from A's partial fix.  
**Fix:** Hard reset to task seed commit.

### Victory lap on tokens
**Symptom:** "Loops are always faster/cheaper."  
**Fix:** Show cases where B costs more but wins on pass rate.
