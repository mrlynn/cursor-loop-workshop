## Thesis (one line)

**A loop = prompt + verifier the agent can iterate against.**

## 15-minute path

1. `./scripts/copy-harness.sh /path/to/lab-repo` — install rules, skills, commands
2. `./scripts/task-prompt.sh oms-status-race-001 --copy` — prompt to clipboard
3. Open lab repo in Agent mode → paste → let the verifier close the loop
4. `./scripts/loop-harness.sh --task oms-status-race-001 --method B --log-only ...` — log evidence

Full skip path: `ADVANCED.md` at the workshop repo root.

## Rules — reference with `@`

| Rule | When to `@` |
|------|-------------|
| `loop-discipline` | Every rung 2+ loop thread |
| `loop-termination` | Any task with iteration budget |
| `loop-safety` | Loops that run shell commands |

```text
@loop-discipline @loop-termination @loop-safety
```

Files live in `.cursor/rules/` — copy into your lab repo with `copy-harness.sh`.

## When to loop vs when to stop

| Loop | Do not loop |
|------|-------------|
| Failing test, lint, or build you can run | Tab-sized single-line edit |
| Fix needs 3+ iterations with regression risk | Pure exploration — use Plan |
| You want A/B evidence vs prompt | No machine-checkable done |

## Honest trade-offs (advanced users expect this)

Loops often **cost more tokens** and sometimes **more wall-clock**. They win on **human interventions** and **pass rate** when the verifier encodes real pain. Do not advocate loops without reporting cost.
