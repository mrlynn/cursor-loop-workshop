Capstone demonstration: same task, arm A then arm B, metrics on screen.

## Why this matters

This is the user story that sells Loop Engineering to skeptics. The learner must *feel* being the feedback loop in arm A before arm B hands it to the verifier.

## What you'll learn

- You can facilitate arm A without verifier leakage
- You can reset seed between arms fairly
- You can present side-by-side JSON in under 5 minutes
- You can lead discussion on "which of your tasks has a verifier?"

## How it works

Use `/run-comparison` skill procedure. This lab compresses it into a paired exercise:

- **Participant A:** runs prompt-only path, counts interventions
- **Participant B:** runs agent loop, counts iterations
- **Pair:** compare logs; facilitator shows cohort aggregate if available

Discussion questions:

```text
1. How many times did a human run the verifier in arm A?
2. Did arm B cost more wall-clock? More tokens?
3. Name one task on your current sprint that has a machine-checkable done.
4. What verifier would you attach?
```

## Common mistakes

### Skipping reset
**Fix:** `git checkout . && git clean -fd` at seed commit.

### Arm A with Agent mode
**Fix:** Arm A uses Chat/Inline — not Agent.

### Declaring victory without cost
**Fix:** Show token/wall-clock even if unfavorable.
