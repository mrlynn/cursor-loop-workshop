Most developers adopting Cursor plateau at **prompt mode**: they ask, read the output, judge it themselves, and ask again. They are the feedback loop. The step-change in value comes when you hand that loop to the agent — goal in, agent acts, agent reads a machine-checkable signal, corrects, and repeats until done.

This workshop teaches the **Loop Engineering Framework**: a reusable substrate, harness, and measurement layer for agentic loops. You will implement a loop, attach a verifier, encode discipline in rules and skills, and compare loop vs. prompt on the same task.

## Who this is for

- Experienced software developers (3+ years) who use Cursor but still default to one-shot prompting
- Workshop facilitators and SEs delivering Cursor 101, hackathons, or Customer Developer Days
- **Prerequisites:** Cursor installed, Agent mode available, a sample repo cloned (`atlas-oms` or equivalent)

## How to use this guide

This is an **instructor-led, hands-on** workshop (~3–4 hours). Follow the sidebar in order. Each section ends with a checklist and optional knowledge check. Facilitators: use presenter mode (`presentation_layout` pages) for projection.

| Section | Time | Outcome |
|---------|------|---------|
| Getting Started | 30 min | Name the autonomy ladder rung you're on |
| Loop Fundamentals | 45 min | Write a task spec with a verifier |
| Rules & Skills | 45 min | Deploy portable loop discipline |
| The Harness | 30 min | Script a reproducible loop run |
| Measurement | 30 min | Log prompt vs. loop metrics |
| Hands-on Labs | 60 min | Close one loop to green on a seeded task |

## What's in this guide

- **Getting Started** — Walk the autonomy continuum (Tab → Cloud → AI SDLC) and map yourself on the ladder
- **Loop Fundamentals** — Engineer the four surfaces: task, verifier, rules, termination
- **Rules & Skills** — Encode loop discipline so it survives context pressure
- **The Harness** — Compose Agent mode, CLI, and Hooks into a reproducible loop
- **Measurement** — Run A/B/C comparison and log honest metrics
- **Hands-on Labs** — Attach a verifier and close a seeded `atlas-oms` task to green
