Context pressure erodes verbal instructions. Rules and skills make act‚Üíverify‚Üícorrect‚Üírepeat portable across threads, facilitators, and repos.

## Why this matters

FR-6 requires loop discipline portable with minimal edits. Without committed rules, each facilitator improvises prompts and cohorts get incomparable results. Skills package multi-step procedures (run comparison, craft verifier) that should not bloat always-apply token budget.

## What you'll learn

- You can distinguish when to use a rule vs a skill vs a command for loop work
- You can deploy the three workshop loop rules from `.cursor/rules/`
- You can invoke `/loop-engineering`, `/verifier-crafting`, `/run-comparison`
- You can keep always-apply rules lean while encoding non-negotiable discipline

## How it works

| Mechanism | Loop role | Activation |
|-----------|-----------|------------|
| **Rules** | Passive discipline ‚Äî verify after every edit, termination, safety | `@loop-discipline` or globs / always-apply |
| **Skills** | On-demand procedures ‚Äî run a loop, craft verifier, A/B/C comparison | `/loop-engineering` or agent auto-load |
| **Commands** | Manual facilitator shortcuts | `/demo-reset` (if defined) |

Workshop ships:

- `loop-discipline.mdc` ‚Äî act‚Üíverify‚Üícorrect‚Üírepeat
- `loop-termination.mdc` ‚Äî green, blocked, budget
- `loop-safety.mdc` ‚Äî destructive command gates, secret hygiene

Skills:

- `loop-engineering` ‚Äî execute a full loop on a task spec
- `verifier-crafting` ‚Äî design pass/fail verifiers
- `run-comparison` ‚Äî A/B/C measurement procedure

Rule of thumb: **if you've corrected the agent three times on the same thing, make it a rule. If it's a multi-step procedure invoked occasionally, make it a skill.**

## When to use this

| Need | Rule | Skill | Command |
|------|------|-------|---------|

## Common mistakes

### 50-line always-apply rule
**Symptom:** Every chat pays token tax; team disables rules.  
**Fix:** Short pointed rules; move procedures to skills.

### Skills as rules
**Symptom:** Comparison procedure loads every thread.  
**Fix:** `run-comparison` is a skill ‚Äî facilitator invokes on demand.

### Rules never referenced
**Symptom:** `alwaysApply: false` and never `@` mentioned ‚Äî agent ignores discipline.  
**Fix:** Facilitator template always includes `@loop-discipline` in loop threads.
